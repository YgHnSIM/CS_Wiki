- [[존 폰 노이만]] — EDVAC 보고서를 통해 저장 프로그램 컴퓨터의 논리 구조와 폰 노이만 구조 명칭이 확산되는 데 핵심 역할을 한 수학자. (소스 1개)
- [[그레이스 호퍼]] — FLOW-MATIC을 통해 사람이 읽고 협업할 수 있는 프로그래밍 언어 표현의 중요성을 보여준 초기 언어 설계 인물. (소스 2개)

## 개념 (Concepts)

- [[차분 기관]] — 유한 차분법을 이용해 수학 표 계산과 인쇄를 자동화하려 한 특수 목적 기계. (소스 2개)
- [[해석 기관]] — Mill, Store, 천공 카드, 반복, 조건 분기를 포함한 범용 기계식 계산 장치 설계. (소스 2개)
- [[유한 차분법]] — 다항식 계산을 반복 덧셈으로 환원해 기계식 계산에 적합하게 만드는 방법. (소스 2개)
- [[알고리즘적 사고]] — 문제를 단계, 데이터, 반복, 조건, 관찰, 재사용 가능한 단위로 분해해 기계가 수행할 수 있게 만드는 사고방식. (소스 4개)
- [[기호 조작]] — 숫자, 데이터, 프로그램 표기를 관계와 의미의 표현으로 보고 규칙에 따라 변환하는 컴퓨팅 관점. (소스 5개)
- [[인코딩]] — 정보를 매체와 목적에 맞는 기호·비트 체계로 변환하는 과정. (소스 1개)
- [[데이터 표현]] — 비트 패턴을 타입과 인코딩 규칙에 따라 의미 있는 데이터로 조직하는 방식. (소스 1개)
- [[이진법]] — 0과 1 두 기호를 사용해 수와 데이터를 표현하는 현대 컴퓨팅의 기본 수 체계. (소스 1개)
- [[비트와 바이트]] — 비트와 8비트 바이트를 중심으로 데이터 표현의 최소 단위와 해석 범위를 정리한 개념. (소스 1개)
- [[ASCII]] — 문자에 숫자 값을 대응시키는 7비트 문자 인코딩 표준. (소스 1개)
- [[유니코드]] — ASCII의 한계를 넘어 다양한 언어 문자와 기호를 포괄하기 위한 문자 체계. (소스 1개)
- [[비트 연산]] — 시프팅과 마스킹으로 특정 비트를 이동·추출·제어하는 저수준 데이터 조작 기법. (소스 1개)
- [[C 문자열]] — Null terminator로 문자열 경계를 표시하는 C 스타일 문자열 표현. (소스 1개)
- [[메모리 안전성]] — 프로그램이 할당된 메모리 범위와 생명주기 안에서 데이터를 다루도록 보장하는 성질. (소스 1개)
- [[정수 오버플로]] — 정수 연산 결과가 타입의 표현 범위를 넘어 안정성과 보안 문제를 일으키는 현상. (소스 1개)
- [[EDSAC]] — 초기 저장 프로그램 컴퓨터로, 소프트웨어의 입력·적재·디버깅·재사용 계층이 선명하게 드러난 사례. (소스 9개)
- [[저장 프로그램 컴퓨터]] — 프로그램 명령을 데이터처럼 메모리에 저장하고 실행하는 컴퓨터 구조. (소스 8개)
- [[EDVAC]] — 저장 프로그램 컴퓨터의 논리 구조를 문서화하고 후속 기계 설계에 영향을 준 초기 전자식 컴퓨터 설계. (소스 1개)
- [[폰 노이만 구조]] — 명령과 데이터를 같은 메모리 체계에 두고 제어 장치가 명령을 가져와 실행하는 컴퓨터 구조. (소스 1개)
- [[Manchester Baby]] — 1948년 저장된 프로그램을 전자식 디지털 컴퓨터에서 실제로 실행한 실험 장치. (소스 1개)
- [[Initial Orders]] — EDSAC에서 종이 테이프 프로그램을 읽고 해석해 메모리에 적재한 초기 로더·어셈블러 절차. (소스 4개)
- [[부트스트랩]] — 작은 절차가 더 큰 절차를 불러오며 시스템을 단계적으로 시작하는 구조. (소스 4개)
- [[로더]] — 프로그램을 실행 가능한 위치에 적재하고 실행 흐름을 넘기는 소프트웨어 계층. (소스 2개)
- [[어셈블러]] — 사람이 읽을 수 있는 기호 명령을 기계 명령으로 조립하는 프로그램. (소스 2개)
- [[자기 수정 코드]] — 프로그램이 실행 중에 자기 명령어나 다른 코드 영역의 명령을 바꾸는 방식. (소스 4개)
- [[재배치]] — 코드가 실제로 메모리에 놓이는 위치에 맞게 주소를 보정하는 과정. (소스 2개)
- [[링커]] — 여러 코드 조각을 결합하고 주소와 참조 관계를 해결하는 소프트웨어 계층. (소스 2개)
- [[디버깅]] — 프로그램 실행을 관찰하고 오류 원인을 찾아 수정하는 소프트웨어 개발의 핵심 활동. (소스 3개)
- [[포스트모템 루틴]] — 프로그램 정지 뒤 메모리 상태를 출력해 오류 원인을 재구성하게 하는 초기 디버깅 도구. (소스 1개)
- [[서브루틴]] — 호출 후 원래 위치로 돌아오는 재사용 가능한 코드 단위. (소스 7개)
- [[Wheeler Jump]] — 호출부 명령을 이용해 복귀 점프를 만들어내는 EDSAC의 초기 서브루틴 호출 기법. (소스 6개)
- [[복귀 주소]] — 서브루틴이나 함수가 끝난 뒤 실행을 재개해야 할 위치를 가리키는 주소. (소스 2개)
- [[스택]] — 복귀 주소와 실행 상태를 LIFO 순서로 저장해 중첩 호출과 재귀를 가능하게 하는 구조. (소스 1개)
- [[스택 프레임]] — 함수 호출마다 스택에 마련되는 매개변수, 복귀 주소, 지역 변수 등의 실행 환경. (소스 1개)
- [[라이브러리]] — 검증된 재사용 코드 단위를 모아 공동체의 소프트웨어 자산으로 만든 체계. (소스 6개)
- [[라이브러리 카탈로그]] — 서브루틴의 동작, 사용법, 실행 시간, 정밀도, 저장 공간을 문서화한 초기 API 문서형 체계. (소스 1개)
- [[API]] — 기능 제공자와 사용자 사이의 동작, 호출 규약, 비용, 전제 조건을 정리한 프로그래밍 인터페이스. (소스 1개)
- [[자동 프로그래밍]] — 주소 보정, 서브루틴 조립, 변환·해석 루틴처럼 프로그램 작성의 규칙적 작업을 기계에 맡기는 초기 기법. (소스 1개)
- [[마이크로프로그래밍]] — 컴퓨터 제어부 동작을 더 낮은 수준의 프로그램처럼 조직하는 설계 방식. (소스 2개)
- [[프로그래밍 언어]] — 인간의 개념적 표현을 기계 실행 절차로 번역하는 표기 체계. (소스 3개)
- [[Fortran]] — 수학적 수식과 과학 계산 표현을 기계 실행 절차로 번역하려 한 초기 고급 프로그래밍 언어. (소스 2개)
- [[소프트웨어 공학]] — 소프트웨어를 신뢰 가능하고 이해 가능하며 유지보수 가능한 구조로 만드는 공학적 실천. (소스 8개)

- [[초기 소프트웨어의 계층화]] — EDSAC에서 드러난 입력, 번역, 적재, 재배치, 호출 상태, 디버깅, 재사용, 언어 설계 계층을 현대 소프트웨어 스택과 비교한 분석. (소스 7개)
- [[비트 패턴과 해석 규칙]] — 비트 자체가 아니라 인코딩, 타입, 경계 조건이 데이터 의미와 보안성을 결정한다는 분석. (소스 1개)
- [[서브루틴 라이브러리에서 API로]] — EDSAC 서브루틴 라이브러리의 카탈로그, 사용 규약, 비용 정보, 오류 진단을 현대 API의 전사로 해석한 분석. (소스 1개)
- [[저장 프로그램 개념의 여러 기원]] — EDVAC, Manchester Baby, EDSAC을 설계·실행·실용 운영 기준으로 나누어 정리한 분석. (소스 2개)
- [[EDSAC은 무엇의 최초인가]] — EDSAC의 최초성을 절대적 최초가 아니라 실용적 범용 저장 프로그램 컴퓨터 기준으로 정리한 분석. (소스 2개)
