## 목적

이 문서는 세션 전환이나 대화 맥락 압축 뒤에도 확장 작업의 순서와 완료 조건을 복원하기 위한 영속 실행 기록이다. 각 우선순위는 검증을 통과하고 원격 브랜치에 푸시된 뒤에만 다음 단계로 진행한다. `raw/` 원본과 `.codex-remote-attachments/`는 작업 범위에서 제외한다.

## 단계별 범위

### P0 — 기존 핵심 관계 보강

새 문서 없이 [[컴퓨팅 능력의 발달사]], [[평균 성능은 왜 서비스의 컴퓨팅 능력을 설명하지 못하는가]], [[인코딩 심화]], [[비트 패턴과 해석 규칙]], [[Unix와 C]], [[Fortran과 컴파일러]], [[병렬 컴퓨팅은 시간을 줄이는가 문제를 키우는가]], [[범용성은 어떻게 컴퓨팅 능력이 되었는가]]에 출처 기반 핵심 관계를 추가한다.

### P1 — 대기열·SLO·에너지

리틀의 법칙과 예측 가능한 DNN 서빙 원전을 참고 자료로 추가하고, 대기열·부하 제어를 LLM 서비스 지표, 연속 배칭, 꼬리 지연, 에너지 비용에 연결한다.

### P2 — 분산 상태·일관성

CAP, Raft, Spanner 원전을 바탕으로 분산 합의·복제 로그·일관성을 기존 결함 허용·가용성·꼬리 지연 축에 연결한다.

### P3 — 입력 해석·보안

입력 검증과 정수 범위 오류를 유니코드 정규화·동형이의 문자·C 문자열·메모리 안전성·API 경계와 함께 분석한다.

### P4 — 벤치마크 측정 계약

SPEC, LINPACK, Green500, MLPerf, TokenPowerBench의 작업·결과·경계·시간·자원 조건을 공통 비교 틀로 종합한다.

### P5 — 검토·출처 보존

`review` 상태 문서를 원전 대조와 관계 보강 뒤 활성화하고, 공개 접근 가능한 중요 `external-only` 자료의 보존 우선순위를 정리한다.

#### 외부 링크 보존 점검 (2026-07-25)

P1의 전체 링크 검사에서 새 대기열·SLO 자료의 USENIX URL은 정상 응답했고, 기존 NIST 참고 자료 두 건은 직접 PDF 경로 소실로 확인됐다. P5에서 원문·기관 대체 자료를 다시 대조해 [[DLMF Standard Reference Tables on Demand]]는 NIST 출판물 레코드로, [[Handbook of Mathematical Functions with Formulas, Graphs, and Mathematical Tables]]는 현재 응답하는 NIST 공식 편람 기록으로 접근 경로를 갱신한다. `raw/` 원본은 이 점검 때문에 변경하지 않는다.

## 재개 절차

1. 이 표에서 처음으로 `진행 중` 또는 `대기`인 단계를 확인한다.
2. `git log`와 완료 커밋 열로 원격 반영 여부를 확인한다.
3. 해당 단계의 범위만 편집하고, 다음 공통 검증을 모두 통과시킨다.
4. 커밋 해시와 검증 결과를 이 표와 [[log]]에 기록한 뒤 푸시한다.
5. 다음 단계의 상태를 `진행 중`으로 바꾼다.

## 공통 검증

- `git diff --check`
