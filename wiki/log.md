Goldberg 논문은 754-1985의 역사적 해설로, 754-2019는 2026년 7월 현재 활성 표준으로 구분했으며 P754는 아직 표준이 아닌 차기 개정 프로젝트로 기록했다.

Goldberg 원 게재본 44쪽과 1991년 정오표를 전수 대조해 정확한 차와 계산된 차의 기호, 수식·예시·극한값·서지 수정 7건을 확인했다. Oracle 공개본은 편집 재수록본이며 뒤의 `Differences Among IEEE 754 Implementations`가 Goldberg 저작이 아니라는 판본 경계도 명시했다. IEEE 표준 전문은 저작권과 접근 범위를 지켜 `raw/`에 복제하지 않고 공식 공개 메타데이터만 `external-only`로 요약했다.

새 개념 [[부동소수점 정확성]]은 표현 불가능한 값, ulp·상대·절대 오차, 가드 숫자와 정확한 반올림, 양성·위험한 상쇄를 구분한다. 비트 단위 재현성, 허용 오차 안의 수치 일치, 과업 의미와 벤치마크 검증을 네 결과 계약으로 분리했다. 새 분석 [[더 빠른 계산은 같은 답을 내는가]]는 컴파일러·병렬 실행이 연산 순서를 바꿀 때 결과 계약을 통과한 작업량만 같은 계산의 성능으로 셀 수 있음을 종합한다.

LINPACK은 64비트 전체 정밀도와 잔차 검증을 요구하지만 IEEE 754 전체 적합성 시험은 아니라고 한정했다. SPEC CPU 2026은 실행 규칙상 IEEE 754를 요구하지 않으며, 출력 검증을 통과하는 일부 산술 재배열을 허용하므로 IEEE 적합성이나 비트 재현성의 증거로 해석하지 않았다. Roofline의 FLOP/s도 정확성 지표가 아니라 유효한 실행 안에서 계산·메모리 병목을 분석하는 값으로 구분했다. 웹에는 17번째 학습 경로 `부동소수점 성능과 정확성`을 추가하고 전체 컴퓨팅 능력 발달 경로에 수치 의미의 표준화 단계를 배치했다.

## [2026-07-16] update | 컴퓨팅 능력 독서 지도와 핵심 분석 시리즈

컴퓨팅 능력 주제의 단계적 보강 가운데 7단계 통합 작업을 수행했다. 새 메타 페이지 [[컴퓨팅 능력 독서 지도]]를 작성해 정의 허브, 핵심 분석, 원전과 기존 17개 학습 경로를 하나의 읽기 체계로 연결했다. 새 선형 경로를 추가하지 않고 빠른 진입, `가능성에서 서비스 능력까지` 핵심 분석 시리즈, 일곱 층위의 17개 원전 코어, 목적별 기존 경로 조합을 분리했다.

원전 코어는 계산 가능성 1편, 알고리즘 복잡도 1편, 프로그래밍 가능성·실현 성능·확장성·자원 효율·신뢰 가능한 결과 각 3편의 `1+1+3+3+3+3+3` 구조로 구성했다. 독자가 각 문서에서 작업, 규모, 결과 계약, 시스템 경계, 고정 조건과 지표를 기록하도록 읽기 완료 기준을 제시하고, 마지막에는 조건이 빠진 “컴퓨팅 능력 두 배” 주장을 비교 가능한 문장으로 다시 쓰게 했다. 후속 조사는 알고리즘 복잡도의 실용적 의미, 결함 허용과 가용성, 근사·혼합 정밀도, 프로그래밍 노력의 측정으로 분리했다.

[[컴퓨팅 능력이란 무엇인가]]에는 핵심 분석 순서를 추가하되 일곱 층위와 분석이 1:1로 대응하지 않음을 명시했다. 알고리즘 복잡도는 연결 절이고, 신뢰 가능한 결과는 수치와 서비스 계약의 두 분석으로 나뉜다. [[컴퓨팅 능력의 발달사]]에는 기존에 빠져 있던 해석 기관, 저장 프로그램, EDSAC 서브루틴·카탈로그, Fortran 컴파일러와 Unix 운영 환경의 전환을 `범용성에서 프로그래밍 가능성으로` 절로 보강했다. 설계·최초 실행·실용 운영, 재배선·외부 설정·내부 기억장치 변경을 구분하고 발달사 근거를 22개에서 27개로 확장했다.

웹의 `컴퓨팅 능력의 발달사` 우산 경로는 독서 지도를 첫 단계로 배치하고 프로그래밍 가능성, Gustafson의 확대 크기 관점과 전력 장벽 분석을 직접 포함하도록 보강했다. 전체 학습 경로 수는 17개로 유지했으며, 메뉴 순서는 더 빠른 프로세서 → 부동소수점 정확성 → 병렬 확장성 → 전력 → 서비스의 논리적 순서로 정렬했다. README의 오래된 10개 경로 표기도 17개와 독서 지도로 수정했다.

새 문서는 `draft → review → active` 순서로 승격했다. 초안과 검토 상태에서 각각 페이지 구조, 출처 매핑, 관련 항목의 상호 링크와 카탈로그 해석을 검사했고, 최종 감사에서 *The Preparation*의 판본 표현, 층위–분석 대응, 발달사 연대 순서와 저장 프로그램 전환의 과도한 일반화를 교정했다.

확장 가능한 지식 그래프 UI의 3단계로 임의의 두 공개 문서를 선택해 연결 경로를 설명하는 지식 지도를 추가했다. 최단 선 하나만 그리지 않고 최대 6단계 안에서 기본 경로와 중간 문서가 다른 대안 두 개를 찾는다. 각 단계는 이동 방향, 원래 기록 방향, 대표 관계의 이유와 같은 문서 쌍에 겹친 다른 관계를 세로형 읽기 경로로 번역한다. URL에는 제목이 아니라 안정 그래프 ID와 렌즈·대안 번호를 저장해 새로 고침, 공유, 뒤로 가기에서도 같은 선택을 복원한다.

현재 공개 그래프에서 본문 언급은 전체 관계의 절반가량이고 공통 원전도 지름길 허브가 되기 쉽다. 기본 `핵심 연결` 렌즈는 관련 항목과 학습 순서를 우선하고 자동 본문 언급과 고차수·원전 중간 노드에는 추가 비용을 둔다. `개념·학습`, `근거 계보`, `본문 언급 포함` 렌즈를 분리했으며, 비용이 높은 직접 관계도 대안에서 사라지지 않게 했다. 숨김 운영 문서는 종점과 경유점에서 모두 제외하고, 문맥 전용 문서는 검색 대상에서는 빼되 설명에 필요한 중간 경유점으로 사용할 수 있다.

## [2026-07-18] update | 계층형 전체 의미 지도

확장 가능한 지식 그래프 UI의 5단계로 203개 공개 문서를 7개 의미 군집과 17개 군집 회랑에서 시작해 군집 내부, 회랑의 실제 문서 관계, 한 문서의 1·2홉 맥락으로 단계적으로 확대하는 전체 의미 지도를 추가했다. 일반적인 점·선 그래프처럼 연결량만 보여 주지 않고 회랑을 선택하면 두 군집을 잇는 실제 관계 최대 40개에 관계 종류, 기록 방향, 자동 추론·편집 관계 여부와 대표 본문 문맥을 보존해 “왜 연결되는가”를 읽게 한다.

처음에는 manifest와 군집 개요만 내려받고 검색이나 문서 초점이 필요할 때 lookup을, 군집·회랑·문서 초점은 선택 시 각각 지연 로드한다. 현재 압축 크기는 첫 화면 약 9KB, 가장 큰 군집 약 74KB 이하, 문서 초점 약 27KB 이하이며 빌드가 각 상한을 검사한다. 문서 초점은 여러 문서를 한 파일에 누적하지 않고 안정 문서 ID별로 최대 56개 노드·180개 관계를 가진 별도 조각으로 생성한다. 5,000개 문서 모의 검사에서도 한 초점 조각의 크기가 문서 수와 함께 커지지 않음을 확인했다.

군집 화면은 관계가 많아져도 최대 240개 노드와 240개 대표 관계만 표시한다. 편집 관계, 근거, 학습 순서와 관련 항목을 균형 있게 우선하고 자동 본문 언급은 소수 표본으로 제한해 연결이 많은 군집이 다시 선의 덩어리가 되는 현상을 줄였다. 노드 모양은 개념·인물·분석·원전/참고 자료를, 선 색과 패턴은 근거·학습·관련/언급 계열을 구분하며 별도 범례를 제공한다.

Canvas는 보조 시각화로 두고 같은 상태의 군집·문서·관계 목록을 기본 HTML로 함께 제공한다. 각 군집에는 자바스크립트 없이도 접근하는 정적 주소와 전체 문서 링크가 있고, 모바일에서는 접을 수 있는 텍스트 목록을 지도보다 먼저 읽는다. URL은 안정 그래프 ID와 군집·회랑·필터를 저장하며 검색은 한글 입력 조합과 표준 combobox/listbox 역할을 보존한다. 정적 군집 주소에서 전체 지도로 돌아오면 루트 주소로 정규화해 새로고침·앞뒤 이동 뒤에도 같은 상태를 유지한다. 새 배포 뒤 이전 조각이 남지 않도록 모든 지연 로드 URL에 콘텐츠 지문을 붙이고 manifest와 버전이 다른 조각은 거부한다. 로드가 실패하면 마지막 성공 화면과 정적 목록을 유지하고 재시도 경로를 제공한다.

`graph_visibility: context` 문서는 전역 군집·검색·통계와 자체 초점 조각에서 제외하고, 공개 문서와 직접 연결된 경우에만 그 문서의 1홉 초점 맥락으로 포함한다. 운영 메타 문서가 일반 지식처럼 검색되는 일을 막으면서 관계 설명에 필요한 편집 문맥은 잃지 않는다.

## [2026-07-18] update | 역사 시간축과 인과 전환 렌즈

확장 가능한 지식 그래프 UI의 6단계로 컴퓨팅 능력의 발달을 연도순 목록이 아니라 **한계 → 대응 → 새 능력 → 다음 제약**의 이동으로 읽는 역사·인과 렌즈를 추가했다. 출판 연도와 사건 기간을 분리하고 제출·출판, 초판·확인 판본, 관찰 기간의 차이는 `historical_note`에 보존했다. 제목·파일명·위키 작성일에서 연도를 추측하지 않으며, 연도가 없는 문서는 삭제하거나 인접 시대에 끼워 넣지 않고 `연도 미상`에 둔다.

현재 203개 공개 문서 가운데 원전 대조로 시점을 기록한 문서는 28개다. 사건 시점 문서 5개와 출판 시점 문서 23개를 고정된 여덟 연대 구간에 배치하고, 나머지 175개는 아홉 번째 `연도 미상` 구간에 보존했다. 역사 층위는 이론·기계·아키텍처·소프트웨어·시스템·서비스·측정과 미분류 레인으로, 컴퓨팅 능력은 계산 가능성·복잡도·프로그래밍 가능성·실현 성능·확장성·자원 효율·결과 신뢰성으로 교차 필터링한다.

자동 본문 언급이나 관련 항목·근거·학습 순서는 인과로 해석하지 않는다. 편집자가 근거와 설명을 붙인 `responds_to`, `enables`, `precedes`, `constrains`만 승격해 15개 검토 전환을 만들었으며, 데이터 계약상 완전 전환 9개와 누락된 역할을 그대로 보이는 부분 전환 6개로 나뉜다. 같은 대응 문서의 `responds_to`와 `enables`는 한계·대응·새 능력으로 합성하고, 부분 전환에 관계를 보강해도 전환 ID와 공유 URL을 유지한다. 순서나 제약 관계는 의미상 출발 역할의 시점·위치를 따르는 독립 전환으로 보존한다. 알려진 연대가 관계의 기대 순서와 어긋나도 간선을 바꾸지 않고 충돌 상태와 원래 기록 방향을 남긴다.

첫 화면은 manifest와 시대 개요만 불러오고 조회 버킷과 시대 조각은 선택할 때 지연 로드한다. 아홉 시대는 현재 10개 조각이며 한 조각은 최대 120개 문서와 96개 전환 요약으로 제한한다. 직접 선택은 최대 96레코드인 결정적 조회 버킷 하나를 읽고, 전체 검색은 현재 네 버킷을 제한된 동시성으로 합친다. 고차수 전환은 최대 6개 대표 간선·32KiB 요약과 최대 48레코드·64KiB 상세 쪽으로 분리해 한 쪽씩 불러오고 캐시한다. 시대의 첫 조각과 이후 조각에는 각각 복원 가능한 정적 주소가 있고, 자바스크립트가 없어도 해당 문서 링크·쪽 이동과 대응 문서의 전체 관계 표를 사용할 수 있다. 앞으로 공개 문서가 추가되면 안정 ID에 따라 정확히 한 시대 조각으로 자동 수용되고, 상한을 넘는 시대·조회·상세 데이터에만 새 조각이 생긴다.

## [2026-07-18] update | 문서·근거 계보 렌즈

확장 가능한 지식 그래프 UI의 7단계로 원자료와 재현 정보, 정규 소스·참고 자료, 지식 문서와 검토 관계를 방향이 있는 3레일로 읽는 근거 계보 렌즈를 추가했다. 현재 공개 지식 문서 146개와 근거 문서 57개 사이의 문서 단위 `supports` 관계 528개를 빠짐없이 보존하고, 관계 표에 설명과 직접 근거가 함께 기록된 검토 관계 24개를 별도 층위로 분리했다.

화면과 데이터 계약에는 의미 경계를 명시했다. 프론트매터 `sources`는 문서 전체의 등록 근거 묶음이며 문장별 진위나 신뢰도를 뜻하지 않는다. 소스 페이지의 `primary_sources`와 `supporting_sources`는 그 자료 페이지가 기록한 원자료와 보조·접근 자료의 재현 계보로만 표시한다. 근거 수를 신뢰도·독립성 점수로 바꾸지 않고, 같은 자료를 등록한 문서도 합의나 동일 결론으로 해석하지 않는다. 로컬 원본 6개, 보존 스냅샷 7개와 외부 링크 의존 44개는 품질 순위가 아니라 다시 확인할 조건으로 보여 준다.

각 지식 문서, 근거 자료와 검토 관계에 정적 계보 주소를 생성하고 실제 문서 링크와 쪽 이동을 HTML에 보존했다. 자바스크립트는 이 정적 화면을 향상해 문서·자료·관계 검색과 보존 상태 필터를 제공한다. 검색은 전체 조회 데이터를 합치지 않고 NFKC·한국어 소문자 정규화한 제목·별칭의 앞 2–4글자에 해당하는 결정적 prefix 버킷만 한 쪽씩 요청한다. 한글 조합 중에는 요청하지 않으며 표준 combobox/listbox 역할, 고유 option ID, 방향키·Enter·Escape와 정상 Tab 이동을 지원한다.

근거 데이터는 manifest, 제한된 개요, 직접 조회, prefix 검색, 문서·관계 초점과 근거 역색인으로 분할했다. 조회·검색은 최대 96레코드와 64KiB, 초점 미리보기는 최대 8레코드와 32KiB, 전체 상세는 최대 48레코드와 64KiB를 동시에 지킨다. 문서나 자료 하나에 1,000개 연결이 생기는 합성 데이터에서도 연결을 정확히 한 번씩 여러 쪽으로 나누고, 한 원자 레코드가 상한을 넘으면 자르지 않고 빌드를 실패시킨다. 브라우저는 하나의 제한 LRU에 현재 검색 조각만 보존하므로 문서 수가 늘어나도 상세 데이터를 무제한 누적하지 않는다.

미래 문서 증가를 전제로 한 독립 감사 뒤 정적 화면의 확장성 경계도 강화했다. 근거·하류 문서뿐 아니라 검토 관계도 초점마다 12개씩 정확히 한 번 렌더링하고, 전체 쪽 번호를 매번 늘어놓지 않고 첫 쪽·마지막 쪽과 현재 앞뒤 두 쪽만 보여 준다. 직접 사용된 `context` 자료는 공개 문서나 검토 관계에서 따라갈 정적 주소와 역색인을 만들지만 전역 검색·통계·대표 진입점·사이트맵에는 넣지 않는다. 관계 표에 편집자가 쓴 `supports`는 검토 관계로만 남기고 프론트매터에서 파생된 문서 단위 근거와 섞지 않는다. 개별 소스 페이지의 `primary_sources` 표기도 주장 강도처럼 읽힐 수 있는 ‘직접 근거’에서 재현 계보를 뜻하는 ‘원자료로 기록’으로 통일했다.

위키 lint도 보강했다. 관계 표의 근거는 반드시 `type/source` 또는 `type/reference` 페이지여야 하며, 같은 문서의 관계 종류·대상이 별칭을 거쳐 중복되는 행을 거부한다. 공개 일반 문서의 `sources`는 위키 소스·참고 자료에 유일하게 해석돼야 하고, 소스 페이지 자체의 raw 파일명 목록은 기존 재현성 계약을 유지한다.
