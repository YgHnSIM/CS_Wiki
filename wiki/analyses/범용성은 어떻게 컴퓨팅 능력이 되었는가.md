## 개요

컴퓨터의 범용성은 “여러 계산을 할 수 있다”는 추상적 성질만으로 완성되지 않았다. 계산 절차를 물리적 장치에서 분리해 기호로 표현하고, 메모리에 저장하고, 자동으로 적재·번역하며, 다른 프로그램과 기계에서 재사용할 수 있어야 범용성은 실제 [[컴퓨팅 능력이란 무엇인가|컴퓨팅 능력]]이 된다.

이 과정의 핵심 변화는 새로운 산술 연산을 계속 추가한 것이 아니다. **기계에 일을 시키는 비용과 변경 단위를 낮춘 것**이다. 기계 구조를 바꾸던 단계에서 메모리의 명령을 바꾸는 단계로, 다시 고급 언어의 소스와 재사용 가능한 소프트웨어 환경을 바꾸는 단계로 이동했다.

“범용성이 컴퓨팅 능력이 되었다”는 표현은 개별 원전이 직접 제시한 용어가 아니라 여러 시대의 자료를 연결한 이 분석의 해석 틀이다. 특히 보편 튜링 기계와 저장 프로그램 컴퓨터 사이에는 개념적 유사성이 있지만, 여기서는 전자가 EDVAC이나 EDSAC에 직접 영향을 주었다는 인과 계보를 주장하지 않는다.

## 특수 목적 자동화와 범용 절차

[[컴퓨팅의 기원 - 배비지와 러브레이스]]에서 [[차분 기관]]은 유한 차분법으로 [[수학 표]] 계산과 인쇄를 자동화하는 특수 목적 장치다. 특정 계산을 빠르고 오류 없이 반복하는 능력은 있었지만, 계산 종류를 바꾸는 유연성은 제한되었다.

[[해석 기관]]은 Store와 Mill, 천공 카드, 반복과 조건 분기를 통해 프로그램에 따라 여러 절차를 수행하는 범용 장치를 구상했다. 이 설계의 중요성은 현대 컴퓨터와 물리적으로 같다는 데 있지 않다. 기계의 기능을 고정된 기구가 아니라 외부에서 주어지는 명령 순서가 결정한다는 관점을 제시했다는 데 있다.

에이다 러브레이스의 해석은 계산 대상을 수량에 한정하지 않고 규칙에 따라 표현할 수 있는 기호 관계로 확장했다. 범용성은 많은 산술 기능의 목록이 아니라, 기호화된 대상과 절차를 같은 기계적 규칙으로 처리할 수 있다는 관점에서 출발한다.

## 보편성: 기계의 기술도 입력이 된다

[[On Computable Numbers with an Application to the Entscheidungsproblem]]의 [[보편 튜링 기계]]는 다른 튜링 기계의 기술을 입력으로 받아 그 동작을 모사한다. 특정 계산마다 새로운 종류의 기계를 정의할 수 있지만, 그 정의 자체를 기호열로 부호화하면 하나의 보편 기계가 여러 기계의 역할을 맡을 수 있다.

이론적 보편성은 컴퓨터의 정체성을 뒤집는다. 기계가 무엇을 하는가는 장치의 고정된 이름보다 주어진 기술과 입력에 달려 있다. 그러나 보편 튜링 기계는 실제 컴퓨터의 사용 편의성이나 속도를 보장하지 않는다. 가능한 계산의 범위와 그 범위에 접근하는 [[프로그래밍 가능성]]은 구분해야 한다.

## 저장 프로그램: 재배선에서 메모리 변경으로

[[First Draft of a Report on the EDVAC]]는 명령과 수치 데이터를 같은 고속 메모리의 기호적 대상으로 다루는 구조를 체계적으로 유통시켰다. 새 작업을 위해 배선을 다시 구성하는 대신 메모리에 다른 명령열을 넣는 방식은 프로그램 변경의 물리적 비용과 시간을 크게 낮춘다.

[[The Manchester Small Scale Experimental Machine - The Baby]]는 1948년 저장된 프로그램의 전자적 실행을 입증했다. 하지만 원리 증명만으로 실용적 범용 컴퓨팅 환경이 완성된 것은 아니다. 메모리에 프로그램을 넣고 주소를 맞추며 오류를 찾아내는 절차가 여전히 필요했다.

이 차이는 [[저장 프로그램 개념의 여러 기원]]이 설계·최초 실행·실용 운영을 분리하는 이유이기도 하다. 범용성은 한 번의 발명보다 설계 원리, 작동 기계와 운영 소프트웨어가 차례로 결합한 결과다.

## 초기 소프트웨어: 프로그램 변경 비용을 낮추다

[[초기 소프트웨어의 탄생]]과 [[초기 소프트웨어의 계층화]]는 EDSAC의 [[Initial Orders]], 로더, 재배치, 진단 루틴과 서브루틴을 프로그램 준비 책임의 분화로 설명한다. 저장 프로그램 컴퓨터에서도 사람이 모든 주소와 입력 형식을 직접 관리한다면 작업 변경 비용은 여전히 높다.

Initial Orders는 종이 테이프의 기호 명령을 메모리에 적재 가능한 내부 명령으로 바꿨다. 서브루틴과 Wheeler Jump는 이미 작성한 계산과 복귀 절차를 다시 사용하게 했다. 프로그램을 바꾸는 일은 점차 “처음부터 모든 명령을 작성하는 일”에서 “기존 구성 요소를 조립하고 매개변수를 제공하는 일”로 이동했다.

[[The Preparation of Programs for an Electronic Digital Computer]]의 라이브러리 카탈로그는 루틴의 동작, 호출 방법, 시간, 저장 공간과 정밀도를 문서화했다. 이는 범용 기계의 잠재적 기능을 사용자가 선택할 수 있는 기능 집합으로 바꾸었다. [[서브루틴 라이브러리에서 API로]]가 보여주듯 문서화된 인터페이스는 코드 자체와 함께 컴퓨팅 능력을 전달한다.

## 컴파일러: 문제 표현과 기계 실행을 연결하다

[[FORTRAN Automatic Coding System for the IBM 704 EDPM]]과 [[The FORTRAN Automatic Coding System]]은 수학에 가까운 [[소스 프로그램]]을 IBM 704의 [[목적 프로그램]]으로 번역했다. 고급 표기를 허용하는 것만으로는 충분하지 않았고, 숙련된 기계어 프로그래머에 가까운 실행 효율을 제공해야 과학 계산 사용자가 이를 받아들일 수 있었다.

[[Fortran과 컴파일러]]의 역사적 의미는 추상화와 성능이 반드시 반대라는 가정을 약화한 데 있다. 컴파일러가 제어 흐름과 인덱스 레지스터 사용을 분석하고 기계별 세부를 맡으면서, 더 많은 사용자가 하드웨어 성능에 접근할 수 있었다. 기계의 잠재적 연산률이 문제 영역의 실제 작업 능력으로 전환된 것이다.

## 운영 환경과 이식성: 범용성의 단위를 넓히다

[[The UNIX Time-Sharing System]]은 파일, 프로세스, 시스템 호출, 셸과 파이프를 하나의 조합 가능한 환경으로 제공했다. 사용자는 개별 프로그램뿐 아니라 여러 도구를 연결해 새로운 작업 절차를 구성할 수 있었다. 범용성의 단위가 단일 명령열에서 상호 운용하는 프로그램들의 환경으로 확장되었다.

[[Portability of C Programs and the UNIX System]]과 [[Unix와 C]]는 기계 의존 부분을 제한하고 공통 언어와 인터페이스를 사용해 프로그램과 운영체제를 다른 하드웨어로 옮기는 과정을 다룬다. 프로그램이 한 기계의 수명과 명령 집합에 묶이지 않게 되면서, 소프트웨어에 축적된 능력을 새로운 기계에서도 다시 사용할 수 있게 되었다.

## 범용성의 역설

범용성은 모든 작업에서 최고 효율을 뜻하지 않는다. 범용 프로세서와 소프트웨어 계층은 다양한 작업을 받아들이는 대신 해석, 번역, 제어와 데이터 이동 비용을 지불한다. 특정 작업에서는 전용 회로나 [[도메인 특화 가속기]]가 더 높은 성능과 에너지 효율을 제공한다.

그렇다고 특화가 범용성을 대체하는 것도 아니다. 오늘날 시스템은 범용 CPU와 운영체제가 작업을 조직하고, 컴파일러와 런타임이 적합한 계산을 가속기로 보내는 방식으로 구성된다. 특화 장치의 효율도 범용 소프트웨어 환경 안에서 접근하고 조합할 수 있을 때 실제 능력이 된다.

## 종합: 능력은 변경 비용을 낮춘 계층에 축적된다

| 전환 | 이전의 변경 단위 | 새 변경 단위 | 축적된 능력 |
|---|---|---|---|
| 특수 목적 장치 → 프로그램 기계 | 기계 구조 | 카드와 명령 순서 | 여러 절차의 실행 |
| 개별 기계 → 보편 기계 | 기계 종류 | 기계의 기호적 기술 | 계산 절차의 일반 모델 |
| 재배선 → 저장 프로그램 | 회로 연결 | 메모리의 명령 | 빠른 작업 전환 |
| 기계어 → 로더·라이브러리 | 전체 주소와 명령열 | 재사용 루틴과 규약 | 검증된 계산의 조합 |
| 기계어 → 고급 언어 | 하드웨어 세부 | 문제 영역의 소스 | 작성 생산성과 자동 최적화 |
| 단일 기계 → 이식 가능한 환경 | 기계별 시스템 전체 | 공통 언어와 인터페이스 | 소프트웨어 능력의 이전 |

컴퓨터라는 기계의 역사에서 범용성은 처음부터 완성된 속성이 아니었다. 이론은 하나의 기계가 여러 계산을 표현할 수 있음을 보였고, 저장 프로그램 구조는 작업 변경을 메모리 조작으로 만들었다. 로더·라이브러리·컴파일러·운영체제와 이식성은 그 잠재력을 인간이 감당할 수 있는 비용으로 사용할 수 있게 했다.

따라서 컴퓨팅 능력의 발전은 연산 장치가 빨라진 역사만이 아니다. **같은 기계에 새로운 계산을 부여하고, 이미 만든 계산을 재사용하며, 축적된 소프트웨어를 다음 기계로 옮기는 비용이 낮아진 역사**이기도 하다.
