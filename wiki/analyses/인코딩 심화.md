## 핵심 주장

[[인코딩 심화]]의 핵심은 "문자"라는 말이 하나의 단위를 뜻하지 않는다는 점이다. 텍스트 처리에서는 추상 문자, [[코드 포인트]], [[Unicode scalar value]], [[코드 유닛]], 바이트 시퀀스, [[그래핌 클러스터]], 글리프가 서로 다른 층위에 있다. 이 층위를 섞으면 문자 수 계산, 문자열 비교, 파일 입출력, 보안 검증에서 오류가 생긴다.

## 계층별 모델

| 층위 | 질문 | 예시 |
|---|---|---|
| 추상 문자 | 어떤 문자 개념을 표현하는가 | 라틴 대문자 A, 한글 음절, 결합 부호 |
| 코드 포인트 | Unicode 공간의 어떤 번호인가 | `U+0041`, `U+AC00` |
| [[Unicode scalar value|scalar value]] | UTF 인코딩 대상인가 | surrogate 제외 |
| 코드 유닛 | 인코딩 형식의 저장 단위는 무엇인가 | UTF-8 바이트, UTF-16 16비트 단위 |
| 바이트 시퀀스 | 파일·네트워크에서 어떤 바이트인가 | `41`, `EA B0 80` |
| 정규화 | 동등한 문자열을 어떤 형식으로 맞출 것인가 | NFC, NFD, NFKC, NFKD |
| 그래핌 클러스터 | 사용자가 문자 하나로 보는 단위는 무엇인가 | 결합 문자, 이모지 시퀀스 |
| 보안 경계 | 같아 보이거나 다르게 해석될 위험은 없는가 | 동형이의 문자, 문자 깨짐 |

## UTF-8과 UTF-16의 차이

[[UTF-8]]은 바이트 단위의 가변 길이 인코딩이다. ASCII 범위는 기존 바이트와 그대로 맞고, 나머지 [[Unicode scalar value|scalar value]]는 2-4바이트로 표현된다. 바이트 순서 문제가 없어 네트워크와 파일 교환에 적합하지만, 바이트 수가 문자 수와 같다는 가정은 깨진다.

[[UTF-16]]은 16비트 코드 유닛 기반이다. BMP의 많은 문자는 코드 유닛 하나로 표현되지만, U+10000 이상의 문자는 surrogate pair 두 코드 유닛이 필요하다. 따라서 UTF-16 문자열 길이는 코드 포인트 수나 사용자 지각 문자 수와 다를 수 있다. 또한 바이트열로 저장할 때 big-endian과 little-endian 문제가 있어 [[바이트 순서 표식]]이나 상위 프로토콜이 필요할 수 있다.

## 같은 글자와 같은 바이트는 다르다

Unicode에서는 같은 사용자 지각 문자나 의미상 같은 문자열이 여러 코드 포인트 시퀀스로 표현될 수 있다. 예를 들어 precomposed 문자와 기본 문자 + 결합 부호 시퀀스가 같은 글자처럼 보일 수 있다. 이때 바이트 단위 비교는 실패할 수 있으므로 [[유니코드 정규화]]가 필요하다.

그러나 정규화가 모든 문제를 해결하지는 않는다. 정규화된 문자열도 [[동형이의 문자]] 문제를 가질 수 있다. 예를 들어 서로 다른 문자 체계의 글자가 비슷하게 보이면, 사용자는 같은 이름이나 도메인으로 오인할 수 있다. 따라서 식별자와 보안 경계에서는 정규화, 허용 문자 범위, script mixing 정책, 표시 정책을 함께 다뤄야 한다.

## 디코딩은 정책이다

바이트를 문자열로 바꾸는 과정은 단순 변환이 아니라 정책 선택이다. 잘못된 UTF-8을 만나면 즉시 실패시킬 수도 있고, replacement character로 대체할 수도 있다. 웹에서는 기존 콘텐츠 호환성 때문에 BOM, HTTP 헤더, HTML 선언, 브라우저 알고리즘이 실제 해석에 영향을 준다.

따라서 안정적인 시스템은 다음 순서를 명확히 해야 한다.

- 입력 경계에서 허용 인코딩을 정한다.
- 바이트를 한 번만 디코딩하고, 내부 표현을 정한다.
- 잘못된 바이트 시퀀스의 처리 방식을 정한다.
- 비교와 저장 전 정규화 정책을 정한다.
- 사용자에게 보이는 길이 제한은 그래핌 클러스터 기준인지 명시한다.
- 식별자에는 동형이의 문자와 혼합 스크립트 정책을 적용한다.

## 이 위키에서의 연결

기존 [[데이터 표현과 인코딩]]은 인코딩을 매체와 목적에 맞는 표현 규칙으로 설명했다. 이 분석은 그 관점을 텍스트 처리 내부로 확장한다. [[비트 패턴과 해석 규칙]]이 "비트 자체가 의미를 갖지 않는다"는 점을 강조했다면, 인코딩 심화는 "문자열 자체도 단일한 층위가 아니다"라는 점을 강조한다.
