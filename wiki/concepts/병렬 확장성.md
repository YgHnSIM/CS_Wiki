## 개요

[[병렬 확장성]](parallel scalability)은 프로세서, 코어 또는 노드 같은 병렬 자원을 늘릴 때 시스템이 고정된 문제를 얼마나 더 빨리 풀거나, 정해진 시간 안에 얼마나 더 큰 문제를 풀 수 있는지를 나타내는 성질이다. 자원 수와 최고 연산률만 세는 대신 작업량, 실행 시간과 효율이 함께 어떻게 변하는지를 묻는다.

확장성을 말하려면 먼저 무엇을 고정하는지 밝혀야 한다. [[Validity of the Single Processor Approach to Achieving Large Scale Computing Capabilities]]에서 유래한 암달 관점과 [[Reevaluating Amdahl's Law]]의 Gustafson 관점은 같은 병렬 시스템을 서로 다른 기준으로 평가한다.

## 고정 크기 확장

고정 크기 확장(fixed-size scaling)은 전체 문제와 입력을 그대로 두고 병렬 자원 수 `N`을 늘려 실행 시간 `T(N)`을 줄이는 문제다. 가속비와 병렬 효율은 다음처럼 정의할 수 있다.

`S_fixed(N) = T(1) / T(N)`

`E_fixed(N) = S_fixed(N) / N`

병렬화 가능한 비율을 `p`, 순차 비율을 `s = 1 - p`라 하고 통신 같은 추가 비용을 무시하면 암달의 모델은 다음 상한을 준다.

`S_fixed(N) ≤ 1 / (s + p/N)`

이 관점은 같은 입력의 지연 시간을 줄여야 하는 경우에 적합하다. 그러나 자원을 늘릴수록 프로세서당 맡는 일이 작아지고, 순차 구간과 통신·동기화의 상대적 비중이 커지므로 효율이 낮아질 수 있다.

## 확대 크기 확장

확대 크기 확장(scaled-size scaling)은 병렬 실행 시간을 일정하게 두고 자원 수에 맞춰 문제의 병렬 부분을 키우는 문제다. Gustafson의 모델에서 병렬 시스템의 순차 시간 비율을 `s'`, 병렬 시간 비율을 `p' = 1 - s'`라 하면 다음과 같다.

`S_scaled(N) = s' + Np' = N - (N - 1)s'`

이 값은 같은 입력을 더 빨리 푼 비율이 아니라, `N`개 프로세서가 정해진 시간에 수행한 확대된 작업을 프로세서 한 개로 수행할 가상 시간과 비교한 비율이다. 격자를 더 촘촘하게 하거나 시간 단계를 늘이고, 더 많은 시나리오를 조사하는 계산처럼 추가 자원이 더 큰 문제로 바뀔 때 유용하다.

이 모델에 유한한 고정 가속 상한이 없다고 해서 순차 부분이 사라지는 것은 아니다. 확대 크기 병렬 효율 `S_scaled(N)/N`은 `N`이 커질 때 `1 - s'`에 가까워진다. 순차 비율은 최대 문제 규모의 절대 상한보다 자원 증가당 얻는 효율의 기울기를 낮춘다.

## 현대 확장 용어와의 관계

실무에서 전체 문제 크기를 고정하는 평가는 강한 확장(strong scaling), 프로세서당 문제 크기를 고정해 전체 문제를 늘리는 평가는 약한 확장(weak scaling)이라고 부른다. 고정 크기 모델은 강한 확장과 직접 대응한다. Gustafson의 확대 크기 모델은 약한 확장의 핵심 직관과 가깝지만, 정확히 같은 정의는 아니다. Gustafson은 병렬 실행 시간을 고정하고 순차 시간이 거의 일정하며 병렬 작업이 `N`배가 된다는 첫 근사를 둔다.

따라서 “Gustafson의 법칙은 곧 모든 약한 확장”이라고 등치하기보다, 문제 규모가 자원과 함께 커지는 한 모델로 구분하는 편이 안전하다. NERSC의 실무 정의도 강한 확장은 고정된 전체 문제, 약한 확장은 프로세서당 고정된 문제로 측정한다.

## 비교표

| 구분 | 고정하는 것 | 묻는 질문 | 대표 지표 | 이상적 결과 |
|---|---|---|---|---|
| 고정 크기 | 전체 문제·입력 | 같은 일을 얼마나 빨리 끝내는가 | `T(1)/T(N)`, 효율 | 시간 `1/N` |
| 확대 크기 | 병렬 실행 시간 | 같은 시간에 얼마나 큰 일을 하는가 | `S_scaled(N)` | 작업량 약 `N`배 |
| 강한 확장 | 전체 문제 크기 | 자원 증가에 따라 시간이 어떻게 줄어드는가 | 시간·가속비·효율 | 선형 가속 |
| 약한 확장 | 프로세서당 문제 크기 | 전체 문제가 커져도 시간이 유지되는가 | 시간·처리량 | 일정 시간 |

서로 다른 행의 수치를 하나의 “가속비”로 직접 비교하면 안 된다. 확대 크기 가속이 높아도 고정된 입력의 지연 시간이 크게 줄지 않을 수 있고, 고정 크기 가속이 포화해도 더 큰 문제를 같은 시간에 푸는 능력은 계속 늘 수 있다.

## 이상 모델을 깨는 비용

[[The Landscape of Parallel Computing Research - A View from Berkeley]]가 강조하듯 병렬 하드웨어는 소프트웨어 지원 없이 자동으로 성능을 제공하지 않는다.

- 순차 초기화, 직렬 자료 구조와 전역 결정
- 통신과 동기화, 장벽 대기
- 작업 분배와 부하 불균형
- 공유 메모리·캐시·네트워크 충돌
- 문제 크기 증가에 따른 입출력과 데이터 재배치
- 병렬 알고리즘과 실행 환경의 추가 작업

이 비용들은 자원 수와 문제 크기에 따라 달라진다. 따라서 암달식과 Gustafson식은 예측의 완성본이 아니라 어떤 부분을 고정했고 무엇을 무시했는지 드러내는 기준 모델로 사용해야 한다.

## 측정할 때 기록할 조건

1. 전체 문제 크기와 프로세서당 문제 크기 가운데 무엇을 고정했는가.
2. 실행 시간, 처리한 작업량, 가속비와 효율 가운데 무엇을 측정했는가.
3. 순차 비율의 분모가 직렬 실행 시간인지 병렬 실행 시간인지.
4. 프로세서·코어·노드 수와 메모리·네트워크 구성이 무엇인지.
5. 통신, 동기화, 입출력과 초기화 시간을 포함했는지.
6. 커진 문제의 해상도·정확도·과학적 가치가 비교 가능한지.

## 컴퓨팅 능력에 주는 의미

병렬 확장성은 [[컴퓨팅 능력이란 무엇인가|컴퓨팅 능력]]이 단순히 “같은 일을 더 빨리” 수행하는 능력만은 아님을 보여준다. 더 많은 자원은 지연 시간을 줄일 수도 있고, 같은 시간에 더 정밀한 시뮬레이션이나 더 넓은 탐색을 가능하게 할 수도 있다. 어느 쪽을 능력 향상으로 부르는지는 사용 목적과 고정 조건을 먼저 명시해야 하며, 이 차이는 [[병렬 컴퓨팅은 시간을 줄이는가 문제를 키우는가]]에서 분석한다.
