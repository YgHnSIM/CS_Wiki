## 개요

[[코드 유닛]]은 어떤 인코딩 형식이 실제로 사용하는 최소 저장 단위다. 같은 [[Unicode scalar value]]라도 UTF-8, UTF-16, UTF-32에서 서로 다른 코드 유닛 시퀀스로 표현된다.

## UTF별 코드 유닛

- [[UTF-8]]의 코드 유닛은 8비트 바이트다.
- [[UTF-16]]의 코드 유닛은 16비트 값이다.
- UTF-32의 코드 유닛은 32비트 값이다.

예를 들어 ASCII 범위의 `A`는 UTF-8에서 1바이트, UTF-16에서 16비트 코드 유닛 하나로 표현된다. 반면 U+10000 이상의 문자는 UTF-8에서 4바이트, UTF-16에서 surrogate pair 두 코드 유닛으로 표현된다.

## 실무적 의미

프로그래밍 언어의 문자열 길이 함수가 무엇을 세는지 확인해야 한다. 어떤 언어는 바이트 수, 어떤 언어는 UTF-16 코드 유닛 수, 어떤 언어는 코드 포인트 수를 반환한다. 사용자가 기대하는 "글자 수"는 대개 [[그래핌 클러스터]]에 더 가깝다.
