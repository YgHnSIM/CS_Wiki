## 개요

[[바이트 순서 표식]](BOM, Byte Order Mark)은 텍스트 스트림 앞에 놓여 바이트 순서나 인코딩을 식별하는 데 쓰이는 표시다. Unicode에서는 `U+FEFF`가 이 역할을 하며, UTF-16에서는 big-endian과 little-endian 순서를 구분하는 데 특히 중요하다.

## UTF별 의미

UTF-16과 UTF-32는 코드 유닛이 2바이트 또는 4바이트이므로 저장 순서를 정해야 한다. BOM이 있으면 앞의 바이트 패턴으로 UTF-16BE, UTF-16LE 같은 순서를 구분할 수 있다.

[[UTF-8]]은 코드 유닛이 이미 바이트이므로 바이트 순서 문제가 없다. Unicode 표준은 UTF-8에서 BOM을 요구하지 않는다. 다만 일부 도구와 웹 문맥에서는 `EF BB BF`가 UTF-8 BOM으로 감지될 수 있고, WHATWG Encoding Standard는 호환성 때문에 BOM을 인코딩 라벨보다 우선하는 경우를 정의한다.

## 실무적 주의점

BOM은 파일 시작에서는 힌트가 될 수 있지만, 문자열 데이터 중간에 섞이면 예상하지 못한 문자처럼 동작할 수 있다. 특히 설정 파일, 스크립트, CSV, 프로토콜 헤더처럼 첫 바이트가 의미를 갖는 형식에서는 BOM 처리 정책이 중요하다.

## 관련 항목

- [[UTF-8]] — Unicode scalar value를 1-4개의 8비트 코드 유닛으로 표현하는 ASCII 호환 가변 길이 인코딩.
- [[UTF-16]] — Unicode scalar value를 16비트 코드 유닛 하나 또는 surrogate pair 두 개로 표현하는 인코딩 형식.
- [[코드 유닛]] — UTF-8, UTF-16 같은 인코딩 형식이 문자열을 저장·전송할 때 사용하는 최소 단위.
- [[인코딩 오류]] — 잘못된 바이트열, 부적절한 디코딩, 대체 문자 처리 때문에 생기는 텍스트 해석 오류.
- [[유니코드]] — 다양한 문자와 기호를 코드 포인트, 인코딩 형식, 정규화, 텍스트 경계 문제로 포괄하는 문자 체계.
