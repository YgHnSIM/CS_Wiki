## 개요

[[유니코드 정규화]]는 Unicode 문자열을 특정 동등성 기준에 맞는 표준 형태로 변환하는 절차다. 같은 글자가 하나의 precomposed code point로 표현될 수도 있고, 기본 문자와 결합 부호의 시퀀스로 표현될 수도 있기 때문에, 바이트 단위 비교만으로는 사용자가 같은 문자열이라고 느끼는 값을 놓칠 수 있다.

## 네 가지 형식

- NFC: canonical equivalence를 유지하면서 가능한 경우 합성형으로 만든다.
- NFD: canonical equivalence를 유지하면서 분해형으로 만든다.
- NFKC: compatibility equivalence까지 고려해 합성형으로 만든다.
- NFKD: compatibility equivalence까지 고려해 분해형으로 만든다.

NFC는 일반 텍스트 저장과 교환에서 자주 쓰이는 실용적 선택이다. NFKC와 NFKD는 호환 문자를 더 강하게 접기 때문에 검색, 식별자 비교, 보안 처리에서 유용할 수 있지만, 원래의 시각적·의미적 구분을 잃을 수 있다.

## 주의점

정규화는 보안의 전부가 아니다. 정규화 후에도 서로 다른 문자 체계의 [[동형이의 문자]]가 매우 비슷하게 보일 수 있다. 또한 정규화된 문자열 두 개를 이어 붙인 결과가 항상 같은 정규화 형식을 유지한다고 보장되지는 않으므로, 문자열 결합 후 다시 정규화가 필요할 수 있다.
