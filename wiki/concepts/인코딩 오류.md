## 개요

[[인코딩 오류]]는 바이트열을 잘못된 인코딩 규칙으로 해석하거나, 형식에 맞지 않는 바이트 시퀀스를 디코딩할 때 생기는 문제다. 흔히 문자 깨짐(mojibake)으로 보이지만, 더 근본적으로는 바이트와 문자 사이의 해석 규칙이 맞지 않는 상태다.

## 주요 원인

- 실제 바이트는 UTF-8인데 프로그램이 다른 legacy encoding으로 해석한다.
- legacy encoding 바이트를 UTF-8로 잘못 디코딩한다.
- UTF-8에서 허용되지 않는 byte sequence를 입력으로 받는다.
- BOM, HTTP 헤더, HTML meta, 파일 확장자, 로컬 기본 인코딩이 서로 다른 힌트를 준다.
- 오류를 즉시 실패시키지 않고 replacement character로 대체해 뒤 단계로 넘긴다.

## 보안적 의미

인코딩 오류는 단순 표시 문제가 아니다. 서버와 클라이언트가 서로 다른 인코딩으로 해석하면 필터링, 권한 확인, 경로 처리, 식별자 비교가 엇갈릴 수 있다. 따라서 경계에서는 입력 바이트를 어떤 인코딩으로 받을지 명시하고, 잘못된 시퀀스를 실패 처리할지 대체 처리할지 결정해야 한다.

## 관련 항목

- [[인코딩]] — 정보를 매체와 목적에 맞는 기호·비트 체계로 변환하고, 디코딩 정책과 텍스트 처리 층위를 함께 다루는 과정.
- [[UTF-8]] — Unicode scalar value를 1-4개의 8비트 코드 유닛으로 표현하는 ASCII 호환 가변 길이 인코딩.
- [[UTF-16]] — Unicode scalar value를 16비트 코드 유닛 하나 또는 surrogate pair 두 개로 표현하는 인코딩 형식.
- [[바이트 순서 표식]] — UTF-16·UTF-32 바이트 순서 식별과 일부 텍스트 스트림의 인코딩 신호로 쓰이는 BOM 개념.
- [[유니코드 정규화]] — 같은 문자열처럼 보이는 여러 코드 포인트 시퀀스를 목적에 맞는 동등성 기준으로 맞추는 절차.
