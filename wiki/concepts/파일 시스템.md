## 개요

[[파일 시스템]]은 파일 이름, 디렉터리, 저장 위치, 접근 권한, 장치와의 연결을 관리하는 운영체제 구성 요소다. [[Unix]]에서 파일 시스템은 단순한 저장 장치 관리가 아니라 사용자와 프로그램이 시스템 자원을 다루는 중심 인터페이스가 된다.

## Unix 파일 시스템

Unix 논문은 일반 파일, 디렉터리, 특수 파일을 구분한다. 일반 파일은 시스템이 내부 구조를 강제하지 않는 바이트열에 가깝고, 디렉터리는 이름과 파일을 연결하며, 특수 파일은 장치를 파일처럼 읽고 쓰게 해 준다.

이 설계는 장치 접근과 파일 접근을 같은 `read`/`write` 모델로 다루게 하여 [[시스템 호출]]과 도구 조합을 단순하게 만든다.

## 관련 항목

- [[Unix]] — 파일, 프로세스, 셸, 파이프, 시스템 호출을 중심으로 구성된 Bell Labs의 다중 사용자 대화형 운영체제.
- [[운영체제]] — 하드웨어 자원과 사용자 프로그램 사이에서 파일, 프로세스, 장치, 입출력을 관리하는 시스템 소프트웨어.
- [[시스템 호출]] — 사용자 프로그램이 운영체제 커널의 파일, 프로세스, 파이프, 장치 기능을 요청하는 인터페이스.
- [[유닉스 파이프]] — Unix에서 프로세스 사이의 데이터를 파일 I/O와 비슷한 방식으로 전달하는 통신 채널.
- [[시스템 프로그래밍]] — 운영체제, 컴파일러, 셸, 장치 제어처럼 기반 소프트웨어를 작성하는 프로그래밍 영역.
