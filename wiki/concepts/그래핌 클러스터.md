## 개요

[[그래핌 클러스터]]는 사용자가 문자 하나로 인식하는 단위에 대한 프로그램 가능한 근사다. Unicode Text Segmentation은 extended grapheme cluster를 기본 모델로 제공하며, 텍스트 선택, 커서 이동, 삭제, 문자 수 제한, 정규식 처리에서 중요하다.

## 코드 포인트와의 차이

하나의 [[코드 포인트]]가 항상 사용자가 보는 문자 하나는 아니다. 기본 문자와 결합 부호, 한글 자모 시퀀스, 일부 인도계 문자, 이모지 ZWJ sequence는 여러 코드 포인트가 합쳐져 하나의 그래핌 클러스터를 이룰 수 있다.

반대로 코드 유닛 수는 더 낮은 저장 단위다. 예를 들어 [[UTF-16]]에서는 하나의 [[Unicode scalar value]]가 surrogate pair 두 [[코드 유닛]]으로 표현될 수 있고, 그 scalar value가 다시 다른 코드 포인트와 결합해 하나의 그래핌 클러스터를 이룰 수 있다.

## 실무적 의미

사용자에게 보이는 "글자 수" 제한을 만들 때 바이트 수나 코드 유닛 수만 세면 예기치 않은 결과가 생긴다. 텍스트 편집기에서 Backspace 한 번이 무엇을 지워야 하는지, UI에서 커서가 어디에 놓여야 하는지, 검색에서 한 글자를 어떻게 매칭할지도 그래핌 클러스터 경계와 연결된다.
