## 개요

[[유니코드]]는 다양한 언어 문자와 기호를 포괄하기 위해 등장한 문자 체계이다. 이 소스에서는 [[ASCII]]가 로마자 중심의 7비트 체계라서 다른 언어 문자와 이모지를 표현할 수 없다는 한계의 대응 개념으로 언급된다.

## 의미

유니코드는 문자 데이터가 단순히 바이트 하나와 문자 하나의 대응으로 끝나지 않음을 보여준다. 문자 집합, 코드 포인트, 실제 바이트 인코딩은 구분되어야 하며, 소프트웨어는 텍스트를 다룰 때 문자 수와 바이트 수가 항상 같지 않을 수 있음을 고려해야 한다.

## 내부 모델

Unicode에서 [[코드 포인트]]는 `U+0041`처럼 문자 공간의 위치를 나타내는 값이다. 그러나 모든 코드 포인트가 인코딩 가능한 [[Unicode scalar value]]는 아니다. UTF 인코딩 형식은 surrogate code point를 제외한 scalar value를 [[코드 유닛]] 시퀀스로 바꾼다.

[[UTF-8]]은 scalar value를 1-4바이트로 표현하고, ASCII 범위는 기존 ASCII 바이트와 그대로 맞다. [[UTF-16]]은 16비트 코드 유닛을 사용하며, `U+10000` 이상의 문자는 surrogate pair 두 코드 유닛으로 표현한다. 이 때문에 문자열 길이는 바이트 수, 코드 유닛 수, 코드 포인트 수, 사용자가 보는 글자 수가 서로 다를 수 있다.

## 동등성과 경계

Unicode 텍스트는 같은 글자처럼 보여도 여러 코드 포인트 시퀀스로 표현될 수 있다. [[유니코드 정규화]]는 비교와 검색을 위해 동등한 문자열을 일정한 형식으로 맞추는 절차다. 하지만 정규화는 "사용자가 문자 하나로 보는 단위"를 직접 해결하지 않는다. 편집기 이동, 삭제, 길이 제한에는 [[그래핌 클러스터]] 같은 텍스트 분할 기준이 필요하다.

## 보안적 의미

Unicode는 여러 문자 체계를 함께 다루므로 서로 다른 코드 포인트가 시각적으로 비슷하게 보일 수 있다. [[동형이의 문자]] 문제는 사용자 이름, 도메인, 프로그래밍 언어 식별자에서 스푸핑 위험으로 이어진다. 따라서 보안 경계에서는 정규화, 허용 문자 범위, 혼합 스크립트 정책을 함께 설계해야 한다.

## 관련 항목

- [[ASCII]] — 문자에 숫자 값을 대응시키는 7비트 문자 인코딩 표준이자 UTF-8 ASCII 범위 호환성의 기반.
- [[인코딩]] — 정보를 매체와 목적에 맞는 기호·비트 체계로 변환하고, 디코딩 정책과 텍스트 처리 층위를 함께 다루는 과정.
- [[데이터 표현]] — 비트 패턴을 타입, 인코딩, 텍스트 분할 규칙에 따라 의미 있는 데이터로 조직하는 방식.
- [[비트와 바이트]] — 비트와 8비트 바이트를 중심으로 데이터 표현의 최소 단위와 해석 범위를 정리한 개념.
- [[코드 포인트]] — Unicode 문자 공간에서 특정 위치를 가리키는 숫자 값과 scalar value의 구분을 정리한 개념.
