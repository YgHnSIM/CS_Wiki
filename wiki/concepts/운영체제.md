## 개요

[[운영체제]]는 하드웨어 자원과 사용자 프로그램 사이에서 파일, 프로세스, 메모리, 장치, 입출력 같은 자원을 관리하는 시스템 소프트웨어다. [[Unix]]는 운영체제를 작은 커널, 파일 시스템, 프로세스, 셸, 유틸리티의 조합으로 이해하게 만든 대표적 사례다.

## Unix 사례

Unix 논문은 운영체제의 중심 기능을 계층적 [[파일 시스템]], 프로세스 생성과 제어, 장치 I/O, 사용자 명령 인터페이스로 설명한다. 특히 장치를 특수 파일로 다루고, 프로세스 사이의 통신을 파일 I/O와 비슷하게 다루는 방식은 운영체제 인터페이스를 단순하게 만든다.

## 관련 항목

- [[Unix]] — 파일, 프로세스, 셸, 파이프, 시스템 호출을 중심으로 구성된 Bell Labs의 다중 사용자 대화형 운영체제.
- [[파일 시스템]] — 파일 이름, 디렉터리, 저장 위치, 접근 권한, 장치 연결을 관리하는 운영체제 구성 요소.
- [[시스템 호출]] — 사용자 프로그램이 운영체제 커널의 파일, 프로세스, 파이프, 장치 기능을 요청하는 인터페이스.
- [[유닉스 파이프]] — Unix에서 프로세스 사이의 데이터를 파일 I/O와 비슷한 방식으로 전달하는 통신 채널.
- [[시스템 프로그래밍]] — 운영체제, 컴파일러, 셸, 장치 제어처럼 기반 소프트웨어를 작성하는 프로그래밍 영역.
